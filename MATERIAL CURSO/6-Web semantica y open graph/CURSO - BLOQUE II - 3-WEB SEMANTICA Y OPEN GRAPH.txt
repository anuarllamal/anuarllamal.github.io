1) WEB SEMANTICA
https://www.w3schools.com/html/html5_semantic_elements.asp
Un elemento semantico de HTML es, justamente, uno que tiene SIGNIFICADO. Pero que significa esto? Primero, tenemos que distinguir entre elementos semanticos y no semanticos:

a. Elementos no semanticos: <div>, <span>. No nos dicen nada de su contenido
b. Elementos semanticos: <form>, <article>, etc. DEFINEN el contenido que encierran

Pero esta distincion no acaba aca, sino que lo mas importante se relaciona con 2 otros conceptos mas: 1) la distribucion de los elementos en la web, el layout de las secciones, cada una con su contenido que tiene tal o cual significado; y con lo que encierra para el SEO (el posicionamiento de los robots de Google), para que las busquedas te encuentren mucho mejor.

*Layout*
En la vieja epoca, los sitios se dividian con el contenedor <div>, y se creaban las secciones usando el atributo class para todo. Gracias al Señor, HTML evoluciono, y tenemos esos elementos semanticos para dividir la disposicion de nuestro sitio, como son:
<article> define contenido especifico (posteos, comentarios, articulos). <article> puede ir anidado dentro de <section> Y viceversa
<aside> lo que va al costado del documento
<details>
<figcaption> es el epigrafe de una foto, pero puede ir arriba del <img> o abajo
<figure> indica contenido de ilustraciones, diagramas, fotos, etc. Es decir, <img> va dentro.
<footer> el pie de pagina (puede contener roadmap, sitemap, copyright, informacion del autor o la empresa, links, etc)
<header> la cabecera de otro elemento. Podes tener varias pero no pueden ir una dentro de otra
<main>
<mark>
<nav> encierra un conjunto de enlaces de navegacion
<section> define una seccion (para capitulos, noticias, contacto, introduccion...)
<summary>
<time>

Consejo: conviene armar el layout del sitio en un cuaderno para ver como van a ir acomodadas las cosas antes de meter los elementos en las cajitas. En w3schools te muestran cómo se le puede agregar style mediante css para que quede como la gente, pero despues lo vamos a ver.

ELEMENTO FAVICON - HEAD
El "favicon" es el "icono favorito", es decir, aquel icono que va en la solapa de la pestaña del navegador a la izquierda del titulo del sitio. Esto va dentro del elemento HEAD, que hasta ahora vimos que contenia el <title> nada mas, y se pone asi:

<!DOCTYPE html>
<html>
<head>
  <title>Titulo de la pagina</title>
  <link rel="icon" type="image/x-icon" href="https://www.w3schools.com/favicon.ico">
</head>
<body>

Para insertarlo, utilizamos el elemento <link> que es la vinculacion entre nuestra pagina y otra pagina (un hipervinculo). Con el atributo href lo que introducimos es el enlace de donde viene el icono que ira alli arriba. Es importante? En muchos sentidos, si, porque hace a la distincion de la web. Se pueden crear nuevos u obtener gratis en https://www.favicon.cc/


2) PROTOCOLO OG (OPEN GRAPH)
https://seoalive.com/open-graph/
https://ed.team/blog/que-es-open-graph-y-como-implementarlo
-Es un protocolo desarrollado por Facebook en 2010 que basicamente nos sirve para entender y aprovechar el como los robots de las redes sociales nos leen los metadatos de la pagina para el SEO. Nos sirve para que tambien, al ser compartida nuestra web, vaya con una foto o miniatura ideal que represente el contenido y no pase desapercibida. 
-La estructura de OG depende de la red a la que le apuntamos y se anota con el texto Open graph data como comentario:

<!-- Coloca este código dentro de la etiqueta <head> de tu web -->
 <title>Titulo de Página. 60-70 caracteres como maximo</title>
 <meta name="description" content="Descripcion de pagina. No superar los 155 caracteres." />
 <!-- Twitter Card data -->
 <meta name="twitter:card" value="summary">
 <!-- Open Graph data -->
 <meta property="og:title" content="Titulo" />
 <meta property="og:type" content="article" />
 <meta property="og:url" content=" http://www.example.com" />
 <meta property="og:image" content=" http://example.com/image.jpg" />
 <meta property="og:description" content="Descripcion" />

a. og:title
El título “Title” se encarga de decirle al buscador sobre qué trata la página. Por lo que debe de ser claro, conciso y ser único en cada página. Su máxima extensión son los 67 caracteres.

b. og:description
La “Meta description” será la que describa e informe más extensamente de la página. No debe superar los 155 caracteres y puede contener una frase o CTA que animen al usuario a hacer clic en el resultado. También debe ser única por cada página.

c. og:url:
Se muestra la URL de una página web, pudiéndose mostrar acortada. Intenta utilizar esta etiqueta con una URL lo más limpia posible y única en cada página.

d. og:image
Debes especificar una URL de imagen para que se muestre en cada página. Este es seguramente uno de los grandes olvidados y uno de los más importantes. Google Discover, Google News, Facebook, Twitter, Linkedin… todos ellos cogerán esta imagen como reclamo para el usuario y si es de calidad, podrás conseguir muchísimas visitas. De lo contrario, si dejas el campo vacío, puede que se muestra la foto del autor, un campo vacío… ¡Tu eliges que es mejor! ;)

e. og:video
En el caso de que tengas un vídeo en la página debes especificarlo mediante esta etiqueta que facilita su interpretación.

f. og:type
Esta etiqueta indica en que consiste tu contenido. Si es un artículo, un viseo, un post del blog…Según el tipo que sea podrás asignar unas propiedades u otras.

g. og:site_name
Con ella puedes especificar tu sitio web para que se muestre tu marca.


Completar estas etiquetas CORRECTAMENTE llevara nuestra web hacia los resultados anhelados al poder ser leida y compartida y cotejada correctamente por el algoritmo de las diversas redes:
